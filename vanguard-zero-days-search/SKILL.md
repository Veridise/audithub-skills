# Vanguard Zero Days Search
Identifies the analysis results generated by vanguard for an mpitt batch. Parses the analysis-results and renders the findings. Indentifies the true positive findings by analyzing the corresponding project code bases and summarize the output. 

# Definitions
## mpitt
Multi platform integration testing tool. An internal tool that runs a group of projects and generates a "batch". Each batch is identified by a timestamp. An mpitt batch is essentially a directory. The skill does not need to run mpitt. The batch directory will be an input to the skill. 

## analysis-result
This is output of one run of vanguard on one project for one detector. This is a json file. It contains the the tool and the detector ("analysis-id"). This skill does not need to deep dive into the information in analysis-results

## rmm-cli
A tool that parses an analysis-result object into text form. The output of this tool is to be analyzed by this skill. 

# mpitt batch directory structure
- mpitt batch is an directory. 
- batch directory will contain one directory for each project
- each project direcotry will have a directory named "workarea"
- The workarea contains the source code for the project. 
- The workarea has an directory named "veridise_artifacts"
- The "veridise_artifacts" directory has a directory named "vanguard"
- The subtree of this vanguard directory contains "results" directories. 
- Each result directory contains a json file that is the analysis result for the detector on the project which contains this result.
Note that the "analysis-result" in the json file may not be accurate. The name of the detector that generated the corresponding analysis result should be identified from the name of the parent directory of the result directory that conatins the analysis result json file. 

# Rendering text from analysis-result
rmm-cli can be run using the command `rmm-cli render text <path to analysis result>`



## Roles and routing
Prompt user to ask what role is to be performed

## General expectations

- For a finding being analyzed use only code from the project that contains generated the given finding. 
- Prefer concrete, code-grounded findings over speculation.
